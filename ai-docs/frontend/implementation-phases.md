# 前端实施阶段与验收（产品版）

本文档以产品页面目标为主，不再以 demo 页面拆分为实施出发点。

## Phase 1：应用壳层与导航（必须优先）

目标：先完成产品基础框架，建立统一体验入口。

- 落地左侧菜单壳层：`Home`、`Plugins`、`Devices`、`Settings`。
- 支持侧栏收缩/展开与 CSS 动画。
- 移动端导航降级为抽屉模式。
- 同步统一页面文件命名为 `pages/_xxx/index.vue`（`_home/_plugins/_devices/_settings`）。

验收：

- 四个菜单路由可访问，激活态正确。
- 侧栏收缩动画平滑，展开/收缩无布局抖动。
- 移动端菜单可开关且不遮挡核心内容。

## Phase 2：页面产品稿落地（必须）

目标：四个一级页面达到产品可验收状态。

- `Home`：只显示 `Synra` 与 `version`。
- `Plugins`：搜索 UI + 卡片列表（先不接完整业务逻辑）。
- `Devices`：沿用现有能力但按产品语言重排。
- `Settings`：保持现有诊断与关于结构。

验收：

- `Home` 无 demo 调试信息。
- `Plugins` 可见搜索框、卡片网格、内置插件卡片。
- `Devices` 交互保留且文案产品化。
- `Settings` 功能不回退。

## Phase 3：插件列表体验（必须）

目标：插件列表具备统一视觉规范。

- 插件卡片支持图标回退：
  - 优先 `dist/logo.png`
  - 缺失时回退 UnoCSS icon
- 卡片字段统一：名称、版本、状态。
- 卡片列数在 PC/移动端自适应。

验收：

- 图标缺失场景显示回退图标，不出现空白占位。
- 各断点列数变化合理，无布局溢出。
- 内置 `chat` 插件默认展示为已安装。

## Phase 4：样式语义统一（必须）

目标：统一 UnoCSS 语义色与状态表达。

- 在主题中定义 `primary/success/warning/error/info/surface/muted` 槽位。
- 页面状态提示统一映射语义色，不再局部随意定义。
- 组件颜色使用以语义类为主，减少硬编码十六进制颜色。

验收：

- 连接成功、失败、警告在四个页面颜色语义一致。
- 设计评审可直接依据语义色命名沟通，不依赖具体色值记忆。

## Phase 5：工程化与扩展（后续）

目标：为插件动态页面与路由生命周期做准备。

- 保持自动导入与组件自动注册约定稳定。
- 预留插件页面宿主容器与路由扩展点。
- 增补页面规范到 Cursor 文档体系，保证协作一致。

验收：

- 前端 lint/build 持续通过。
- 新增页面遵循统一目录与命名约定。
- 文档与实现方向一致，无 demo 叙事回流。
