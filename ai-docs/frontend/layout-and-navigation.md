# 布局与导航规范（产品版）

本文档定义产品化页面布局，不以 demo 页块拼接为目标。布局核心是左侧菜单 + 右侧内容区，并保证 PC 与移动端统一语义。

## 断点基线（来自 `apps/frontend/uno.config.ts`）

- `xs`: `320px`
- `sm`: `480px`
- `md`: `768px`
- `lg`: `1024px`
- `xl`: `1280px`
- `2xl`: `1536px`
- `3xl`: `1920px`

## 主布局模型

## 桌面端（`lg+`）

- 左侧固定导航栏：图标 + 名称，支持收缩。
- 右侧为页面主内容容器。
- 推荐骨架：`grid grid-cols-[auto_1fr] min-h-screen`。

## 平板与移动端（`base~md`）

- 左侧导航转为抽屉或覆盖层。
- 顶部保留导航触发按钮（菜单开关）。
- 内容区保持全宽，保证可读性和触达。

## 左侧导航规范

- 菜单项固定四个：`主页`、`插件`、`设备`、`设置`。
- 每项都包含：图标、名称、激活态视觉反馈。
- 收缩态只显示图标；展开态显示图标 + 文本。
- 收缩/展开必须有 CSS 过渡动画（宽度、透明度或位移过渡）。

## 动画规范

- 推荐过渡时长：`180ms ~ 240ms`。
- 推荐缓动：`ease-in-out`。
- 动画只用于状态切换，不干扰内容交互。
- 低性能设备上可降级为仅透明度/位移过渡。

## 页面布局要求

### `Home`

- 内容居中，信息极简。
- 仅展示 `Synra` 与 `version` 一行。

### `Plugins`

- 顶部搜索区 + 下方卡片网格。
- 卡片列数自适应建议：
  - `base/sm`: 1 列
  - `md`: 2 列
  - `lg`: 3 列
  - `xl+`: 4 列（按内容密度可调）

### `Devices`

- 保留设备发现与配对主交互布局。
- 避免把调试字段放在首屏主操作区。

### `Settings`

- 延续现有诊断布局，可卡片化分组展示。

## UnoCSS 颜色语义

建议在 `theme.colors` 维护语义色槽位，并在页面统一使用：

- `primary`：主品牌色、主按钮
- `success`：成功状态（连接成功、安装完成）
- `warning`：待确认或风险提示
- `error`：错误、失败、阻断状态
- `info`：普通信息提示
- `surface`：卡片/面板背景
- `muted`：弱化文本与辅助信息

使用规则：

- 同类状态在所有页面使用同一语义色，不做局部随意替换。
- 错误提示优先 `error`，不要混用 `warning` 造成误判。
- 强调信息优先用 `primary`，避免多种高饱和色同时竞争注意力。

## 多端验收清单

- 桌面端侧栏收缩与展开动画平滑，无布局抖动。
- 移动端抽屉导航可正常开关，且不遮挡核心交互。
- 插件列表在各断点列数变化合理，无卡片挤压溢出。
- 颜色语义在四个页面保持一致（success/error/warning 不混用）。
