# 05 实施路线图

## 总体策略

采用“先协议、再桥接、后能力扩展”的分阶段推进方式。每个里程碑都要求有明确输出物、可验证结果、风险控制与回滚方案。

## 里程碑 M1：工程骨架与协议底座

## 目标

- 将现有模板包改造为可承载 Electron 插件实现的工程结构。
- 固化协议模型、错误码、channel 规范。

## 输出物

- `src/api`、`src/bridge`、`src/host`、`src/shared` 基础目录。
- 协议类型定义与 schema。
- 统一 `invoke` 调用管道的空实现。
- 基础单测（协议解析与错误映射）。

## 验收

- 能通过 `vp check`、`vp test`。
- 有最小 API 可调用并返回 mock 数据。

## 风险与回滚

- 风险：目录调整影响现有导出。
- 回滚：保留 `index.ts` 兼容导出层，内部渐进迁移。

## 里程碑 M2：主通路桥接打通（Preload + Main）

## 目标

- 打通 WebView -> preload -> ipc -> main -> service 的主调用链路。

## 输出物

- preload 暴露层与 main 注册中心。
- 最小 handler 集合（如 runtimeInfo、openExternal）。
- requestId 链路日志。

## 验收

- 最小 demo 可在 Electron 中完成至少 2 个 API 调用。
- 错误场景可返回稳定错误码。

## 风险与回滚

- 风险：安全配置不当导致暴露面过大。
- 回滚：通过单一 `invoke` 入口与白名单机制收敛。

## 里程碑 M3：能力扩展与契约稳定化

## 目标

- 扩展核心能力（文件、系统交互、窗口协作等）。
- 完成协议版本治理与兼容策略。

## 输出物

- 多个稳定 API 与对应 handler/service。
- 协议版本协商机制。
- 契约测试、集成测试覆盖关键路径。

## 验收

- 新增 API 均有类型、schema、测试。
- 向后兼容策略被测试验证。

## 风险与回滚

- 风险：能力扩展导致服务层职责膨胀。
- 回滚：按领域拆 service，禁止跨领域互调。

## 里程碑 M4：性能与发布稳定化

## 目标

- 达成性能基线并具备可发布条件。

## 输出物

- 性能基线报告（冷启动、IPC 延迟、吞吐）。
- 发布前检查脚本与版本策略。
- 文档与示例工程（最小集成样例）。

## 验收

- 性能指标达到 `04-modern-stack-and-performance.md` 目标范围。
- 发布包结构、类型导出、运行验证通过。

## 风险与回滚

- 风险：优化过程中破坏 API 稳定性。
- 回滚：协议版本化 + feature flag 灰度。

## 并行工作建议

- 协议与测试规范可先并行推进。
- 主进程能力适配与 API 封装可按模块并行。
- 所有并行开发分支都需对齐同一协议版本文档。

## 里程碑时间建议（可调整）

- M1：1 周
- M2：1~2 周
- M3：2~3 周
- M4：1 周

## 完成定义（Definition of Done）

- 对外 API、协议、错误码均有文档与测试。
- 关键路径性能满足预算。
- 有可运行 demo，并可在当前 monorepo 工作流下验证。

## 里程碑 M5：跨端通讯链路（LAN 优先 + Relay 回落）

## 目标

- 实现设备间可靠消息传输：先走局域网直连，失败后自动回落中继通道。
- 打通消息 ACK、重试、幂等与会话恢复。

## 输出物

- `@synra/transport-core` 抽象接口与状态机。
- `@synra/transport-lan` 直连实现（发现、连接、保活）。
- `@synra/transport-relay` 回落实现（中继接入、重连）。
- 统一消息追踪字段（`traceId`、`sessionId`、`messageId`）。

## 验收

- 同网段时默认走 LAN，断链后自动切换 Relay。
- 关键消息在弱网场景下可重试并保持幂等。
- 端到端日志可追踪一次动作从手机到 PC 的完整链路。

## 风险与回滚

- 风险：双通道状态一致性复杂，导致重复执行。
- 回滚：所有执行入口先做 `actionId` 幂等校验，重复请求直接回执旧结果。

## 里程碑 M6：插件运行时与执行编排

## 目标

- 建立跨端插件运行时，支撑“分享匹配 -> 候选动作 -> 用户确认 -> PC 执行”。

## 输出物

- `@synra/plugin-sdk`：插件接口、匹配器、配置模型。
- `@synra/plugin-runtime`：插件注册中心、优先级、冲突处理、执行编排。
- 默认示例插件（Github 链接识别并在 PC 浏览器打开）。

## 验收

- 手机端可发出候选动作列表，PC 端可执行选定动作。
- 插件可配置“是否需要确认弹窗”“默认执行目标”。
- 同一分享内容匹配多个插件时，决策行为可预测。

## 风险与回滚

- 风险：插件间冲突导致行为不可预测。
- 回滚：引入优先级 + 显式冲突策略（first-match 或 user-select），并记录决策日志。
