# Synra 主流程路线图

## 总体策略

采用“先协议与插件契约、再通讯链路、后端到端联调与稳定化”的推进方式。每个里程碑都需具备可运行输出和可验证标准。

## M1：协议与插件契约底座

### 目标

- 冻结跨端消息协议和插件接口。
- 完成 `@synra/protocol`、`@synra/plugin-sdk` 最小可用实现。

### 输出物

- 协议类型定义、错误码、消息结构。
- 插件匹配与执行契约类型。
- 基础单测。

### 验收

- 包可构建、可测试。
- 关键类型被主流程文档引用且无冲突。

## M2：通讯链路（LAN 优先 + Relay 回落）

### 目标

- 建立 `@synra/transport-core` 抽象。
- 完成 LAN 主通路与 Relay 回落能力。

### 输出物

- ACK、重试、幂等去重机制。
- 链路追踪字段（`traceId`、`sessionId`、`messageId`）。
- 连接状态可观测。

### 验收

- 同网段默认 LAN，断链自动回落 Relay。
- 弱网场景下消息可重试且不重复执行。

## M3：插件运行时与执行编排

### 目标

- 实现 `@synra/plugin-runtime`。
- 支撑“分享匹配 -> 候选动作 -> 用户确认 -> PC 执行”。

### 输出物

- 插件注册中心与冲突决策策略。
- 标准执行回执与失败重试策略。
- Github 打开插件示例。

### 验收

- 手机端可触发候选动作，PC 端可执行选中动作。
- 多插件匹配时行为可预测并可追踪。

## M4：与 `@synra/capacitor-electron` 集成

### 目标

- 将主流程能力接入 Electron 宿主桥接。
- 打通主链路 demo。

### 输出物

- Electron 侧动作适配器（打开浏览器/文件等）。
- `invoke` 调用链路接入跨端执行结果回执。

### 验收

- 端到端完成“手机分享 -> PC 执行 -> 回执反馈”。

## M5：稳定化与发布准备

### 目标

- 性能与可靠性达标。
- 形成可发布、可回滚的版本策略。

### 输出物

- 主流程性能报告（延迟、切换、重试）。
- 版本发布清单与兼容窗口定义。

### 验收

- 关键指标达到团队设定阈值。
- 文档、包名、导入路径一致性通过检查。
