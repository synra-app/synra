# 产品目标与场景

## 目标

Synra 作为跨设备动作编排系统，要求任意终端都具备统一通信能力：

- 任意节点可发现其他节点，也可被其他节点发现。
- 任意节点可发起连接邀请，也可接收连接邀请。
- 集群内仅一个主机负责中继与插件权威服务。
- 主机故障时自动选举，保证业务链路可持续。

## 统一行为原则

- 设备类型无特权：移动端与客户端均执行同一连接与会话协议。
- 角色按会话赋予：同一节点可在不同会话中充当 `Member` 或 `Host`。
- 消息路径固定：成员间业务消息必须通过主机转发。
- 权威源唯一：插件清单、插件包与规则配置由主机统一提供。

## 核心场景

## 场景 A：移动端触发，客户端执行

1. 移动端发现并连接集群主机。
2. 移动端发送动作请求到主机。
3. 主机转发请求到目标客户端成员。
4. 目标客户端执行后将回执返回主机。
5. 主机聚合并转发结果至移动端。

## 场景 B：客户端触发，移动端执行

1. 客户端发现并连接主机。
2. 客户端向主机发送动作请求。
3. 主机转发给移动端成员执行。
4. 移动端回传执行状态。
5. 主机向客户端回执最终结果。

## 场景 C：主机退出后自动接管

1. 所有成员检测到主机心跳超时。
2. 节点进入选举流程并产生新主机。
3. 新主机发布主机公告并接管中继路由。
4. 成员自动重连并恢复请求通道。
5. 未完成请求按恢复策略补偿或失败回执。

## 非目标

- 本文档不定义云中继实现细节。
- 本文档不定义插件市场与签名平台。
- 本文档不约束 UI 视觉与交互文案。
