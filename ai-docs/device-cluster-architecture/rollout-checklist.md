# 文档落地检查清单

## 目标覆盖检查

- [ ] 已覆盖统一设备行为（移动端与客户端均可发现、被发现、邀请、被邀请）。
- [ ] 已覆盖单主机中继模型与成员路由约束。
- [ ] 已覆盖主机故障后的自动选举与接管规则。
- [ ] 已覆盖主机作为插件与规则唯一权威源的同步链路。
- [ ] 已覆盖会话契约、幂等、重试与错误码。
- [ ] 已明确允许激进式重构，不要求向后兼容旧逻辑。

## 一致性检查

- [ ] 术语仅使用 `Host/Member/Candidate/Term/Epoch/Session/Relay`。
- [ ] 主机相关语义在所有文档中保持一致。
- [ ] 消息类型命名与字段约束在所有文档中一致。
- [ ] 故障码与恢复动作在各章节没有冲突定义。

## 可执行性检查

- [ ] 架构图可直接用于实现拆解。
- [ ] 选举附录可直接转为开发任务与测试用例。
- [ ] 恢复流程可直接映射到故障演练脚本。
- [ ] 指标与事件可直接映射到观测埋点。
- [ ] 已补充 `design-patterns-and-performance-guide.md` 并覆盖全部文档。

## 评审建议

- [ ] 产品评审：场景完整性与体验一致性。
- [ ] 架构评审：单主机中继与选主策略合理性。
- [ ] 客户端评审：会话与重连契约实现可行性。
- [ ] 测试评审：故障注入与恢复验证覆盖度。
