# 插件页面加载与移动端安装

## 统一原则

- 插件页面来自插件 `dist` 构建产物（JS + chunk + CSS）。
- 宿主动态加载插件入口模块并挂载页面组件。
- 不使用 `manifest.json`；元信息来源是插件 `package.json`。

## 页面加载链路（客户端）

1. 拉取插件目录（catalog），获取插件基础信息与入口资源地址。
2. 根据 `pluginId + pageKey` 触发插件进入生命周期。
3. 将插件页面映射到动态路由：`/plugin-${pluginId}/${pageKey}`（例如 `/plugin-chat/home`）。
4. 加载插件入口 JS。
5. 通过 `register(pageKey, loader)` 获取页面加载器。
6. 执行 loader，拿到 `defineComponent` 页面并渲染。

## 移动端“安装插件”定义

移动端安装插件 = 将插件页面产物同步到本地缓存并完成激活注册，而不是应用商店安装行为。

流程建议：

1. 建立与 PC 会话。
2. 请求插件目录，比较本地是否已有同版本插件缓存。
3. 无缓存或版本变更时下载插件 UI 产物。
4. 校验完整性（checksum/hash）。
5. 校验通过后写入缓存目录并标记已安装。
6. 首次进入插件时执行 `onPluginEnter` 注册路由并渲染页面。

## 更新策略

- 以 `package.json.version` 作为版本判断依据。
- 新版本下载成功后原子切换。
- 建议保留最近一个可用版本用于回滚。

## 卸载策略

- 执行 `onPluginExit`。
- 注销插件路由。
- 清理本地缓存目录。

## 安全边界

- 仅加载白名单来源资源（例如受控插件资源服务）。
- 校验失败不激活插件页面。
- 插件页面通过受控桥接 API 与宿主通信，不直接访问底层传输细节。
