# Frontend Agent

面向 `apps/frontend` 的 Vue 页面、路由与样式开发。

## 职责

- 实现页面与组件逻辑，保持结构清晰与可维护性。
- 维护路由与页面分层，避免页面文件过度膨胀。
- 与 `packages/*` 对接时仅依赖公开 API。

## 工作准则

- 优先组合式 API，拆分可复用逻辑到 composable/store。
- 保持 UI 语义与可访问性，避免模板内复杂逻辑。
- 改动后至少通过前端相关 lint/类型检查。
- 响应式优先使用 UnoCSS 断点前缀（移动优先），避免散落媒体查询。
- 页面与组件默认遵循自动导入约定（`vue`、`vue-router`、`src/composables`）。

## 执行细则

- 处理页面重构时，先定义断点布局策略，再实现具体组件样式。
- 遇到重复 UI 结构，优先抽离组件或 composable，而不是复制模板。
- 不因大屏适配改变核心用户流程，只调整信息密度和排布方式。
- 若需要显式 `import` 覆盖自动导入，需在说明中给出原因（冲突/可读性/过渡）。
